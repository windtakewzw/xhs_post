# xhs_post 规则体系设计

## 概述

xhs_post 建立独立的可维护规则体系。规则分为两级：

- **全局规则** (`rules/`)：跨项目共享的"方法论"——怎么写、图片怎么生成、标签怎么打
- **项目规则** (`rules/{项目名}/rules.md`)：每个项目独立配置——用哪几个人设、内容策略、禁忌

规则参照 yj_skills `skills/xiaohongshu/references/` 建立，但独立维护，xhs_post 不依赖 yj_skills 运行时文件。

## 全局规则

### rules/copywriting-rules.md

管辖"怎么写"，跨项目共享，几乎不改。

六大章节：
1. 平台特性 — 小红书 vs 其他平台的差异
2. 标题公式 — 6种模板（数字+场景、疑问+悬念、对比+结论、热点+关联、身份+建议、情绪+场景）
3. 正文结构模板 — 3种（干货清单型、叙事体验型、热点借势型）
4. Emoji 使用规范 — 功能性 emoji 对照表 + 用量限制
5. 去AI味写作法则 — 七条铁律 + 八个AI标志词 + 修改后自查清单
6. 合规红线 — 广告法禁用词 + 房地产合规要求

### rules/image-rules.md

管辖"图片怎么生成"，跨项目共享。

六大章节：
1. 生成模式决策 — 决策树（有实景→img2img、有效果图→img2img、无素材→text2img）
2. 多图排版 — 标准6图结构 + 按内容类型的图片配置表
3. Seedream API 参数 — 端点、认证、模型、尺寸等公共参数
4. 提示词规范 — text2img/img2img 提示词模板结构 + 全局负面提示词
5. 按人设的视觉风格 — 投资顾问(冷色调)/生活顾问(暖色调)/家庭顾问(中性) 三套映射
6. 图片命名与存储 — 文件命名规则 + 输出路径

### rules/hashtag-rules.md

管辖"标签怎么打"，跨项目共享，新增城市时追加。

六大章节：
1. 标签分层策略 — 三层（大流量/精准/长尾），5-8个总计
2. 按城市的标签库 — 海口/三亚/北京/上海/广州/深圳/成都/杭州
3. 按人设的基础标签 — 每个人设的基础包 + 按内容类型追加
4. 按项目特性的补充标签 — 海景/改善/刚需/别墅/TOD
5. 热点标签规则 — 来源、使用限制、负面热点禁用
6. 标签生成检查清单

## 项目规则

### rules/{项目名}/rules.md

每个项目一个文件，定义本项目的具体策略。包含 YAML frontmatter（project/city/type/features）+ 五个章节：

1. **本项目人设** — 每个人设的完整定义（姓名、类型、账号绑定、性格、语言风格、口头禅、写作用语、内容占比、发布频次）
2. **项目特性标签** — 项目名 + 特性标签
3. **项目内容禁忌** — 本项目特定的禁止事项
4. **内容选题序列** — 按人设的发布轮换顺序
5. **发布配置** — 发布窗口、间隔、频次

人设通过 `人设类型` 字段（investment-advisor / lifestyle-advisor / family-advisor）关联到全局规则的视觉风格表和标签模板。

## 规则加载流程

```
生成内容请求（项目名 + 人设 + 内容类型）
    │
    ├─ 1. loader 读取全局三个规则文件
    │     copywriting-rules.md  → 标题公式 + 正文模板 + 去AI味 + 合规
    │     image-rules.md        → 决策矩阵 + 提示词模板 + 视觉风格
    │     hashtag-rules.md      → 城市标签 + 人设标签 + 内容类型标签
    │
    ├─ 2. loader 读取项目规则
    │     rules/{项目名}/rules.md → 人设详情 + 内容禁忌 + 发布配置
    │
    └─ 3. 组装 Prompt 上下文
          系统指令 ← 全局规则的相关章节
          人设设定 ← 项目规则中的人设定义
          内容主题 ← 选题 + 素材
          风格约束 ← 去AI味法则 + 合规红线
```

loader.py 职责：
- 解析 Markdown 章节（按 `## 一、` `## 二、` 分割）
- 提取表格为 list[dict]
- 返回结构化 RuleSet 对象，包含原始文本和结构化数据

## 维护指南

新增城市：在 `hashtag-rules.md` 追加城市标签表
新增人设类型：在三个全局规则文件分别追加相关章节
新增项目：`rules/{项目名}/rules.md` 复制模板后编辑
调整内容占比：编辑项目 rules.md 中的人设内容占比表
调整人设风格：编辑项目 rules.md 中的人设定义
