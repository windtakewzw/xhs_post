系统边界划分 & 集成设计

yj_skills（内容生成）→ xhs_post（发布执行）

两个系统的分工

职责边界
yj_skills（已有，不改）

内容生产方

  • 人设管理（persona-templates.md）
  • 选题决策（营销日历 + 内容类型决策树）
  • AI 文案生成（Claude API + 去AI味规则）
  • 图片生成（Seedream API text2img/img2img）
  • Hashtag 策略（分层标签库）
  • 输出草稿到 drafts/ 目录
xhs_post（新建）

发布执行方

  • 读取 drafts/ 中的待发内容
  • 多账号管理 & 登录态维护
  • Playwright 浏览器自动化发布
  • 发布调度 & 并发控制
  • 风控策略（反检测/行为拟人化）
  • 发布结果回收 & 状态追踪

数据流

端到端流程
① yj_skills 生成
/xiaohongshu-batch 中央半岛 --auto
→ 输出草稿到 data/中央半岛/xiaohongshu/drafts/20260510_001/
→ 每篇包含 post.md（标题+正文+标签+元数据）+ images/（封面+内容图）
⬇
② xhs_post 扫描
调度器定时扫描所有项目的 drafts/ 目录
→ 发现 status: pending_publish 的草稿
→ 匹配对应 account（通过 persona_id 映射）
⬇
③ 发布队列
写入 SQLite posts 表，状态 queued
→ 调度器按账号窗口 + 全局间隔分配发布时间
⬇
④ Playwright 执行发布
加载账号 BrowserContext → 验证登录态 → 上传图片 → 填写内容 → 提交
→ 成功后回收笔记ID，更新 posts 表状态 published
⬇
⑤ 结果回写
更新 drafts/ 中草稿状态 + SQLite 发布记录
→ 更新 account last_published 时间

草稿格式约定（yj_skills → xhs_post 接口）

post.md 元数据格式（在现有基础上增加发布状态）
---
id: draft-20260510-001
project: 中央半岛
persona_id: persona-001
account_id: account-001
content_type: market-analysis
generated_at: 2026-05-10T08:00:00
publish_status: pending         # ← xhs_post 关注的字段
scheduled_at: 2026-05-10T20:00:00  # ← xhs_post 写入
published_at:                # ← xhs_post 回写
xhs_note_id:                   # ← xhs_post 回写
---

yj_skills 生成时写入基础元数据，xhs_post 发布后回写 publish_status / published_at / xhs_note_id

SQLite 表设计（精简版）

表 用途 核心字段
accounts 运行时账号状态 id, account_md_path, login_status, user_data_dir, last_post_at
publish_logs 发布记录 id, draft_path, account_id, status, xhs_note_id, error_msg, published_at
publish_queue 调度队列 id, draft_path, account_id, scheduled_at, priority, retry_count
