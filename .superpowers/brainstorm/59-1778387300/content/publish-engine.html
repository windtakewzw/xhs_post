Playwright 发布流程

单次发布步骤
  1. 创建 BrowserContext — 加载账号专属 userDataDir（含持久化Cookie）
  2. 验证登录态 — 访问小红书首页，检测是否已登录；失效则标记 expired
  3. 进入创作者中心 — 导航到发布页面（处理DOM等待、弹窗关闭）
  4. 上传图片 — 通过文件选择器上传配图（小红书最多18张，房产内容建议3-9张）
  5. 填写标题 — 逐字输入（模拟打字，避免反爬），最多20字
  6. 填写正文 — 分段粘贴，带emoji和换行
  7. 添加话题标签 — 逐个添加，利用平台推荐标签
  8. 设置可见性/位置 — 公开/私密，可选添加地点标签（楼盘位置）
  9. 提交发布 — 点击发布按钮，等待成功回调
  10. 回收笔记ID — 从页面响应/URL中提取新笔记ID，写入数据库

调度策略

20个账号的发布节奏
参数 值 说明
每账号频率 1篇 / 1-2天 20个账号 ≈ 日均10-15篇发布量
发布窗口 8:00-22:00 每个账号设定自己的最佳时段
并发上限 最多3个 Browser 同时运行 避免资源过载 + 降低同一IP并发特征
发布间隔 至少 15 分钟 无论是否同账号，全局最小间隔
同账号间隔 至少 24 小时 避免单账号短时间高频发布
同楼盘去重 3天内不重复主题 同楼盘3-5个账号错开发布主题

# 每个账号独立的持久化上下文
context = await browser.new_context(
    storage_state=f"data/accounts/{account_id}/state.json",
    user_agent="真实的Chrome UA",
    viewport={"width": 390, "height": 844},     # iPhone 14 尺寸，模拟移动端
    device_scale_factor=3,
    is_mobile=True,
    has_touch=True,
    locale="zh-CN",
    timezone_id="Asia/Shanghai",
)

# 反检测脚本注入
await context.add_init_script("""
    // 隐藏 webdriver 特征
    Object.defineProperty(navigator, 'webdriver', {get: () => false});
    // 注入真实的 chrome 对象
    window.chrome = { runtime: {} };
""")
