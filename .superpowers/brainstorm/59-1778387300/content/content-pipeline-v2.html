基于 D:\project\yj_skills\data\ 的目录结构设计

数据源：现有目录 vs 系统数据库 分工

关注点分离
文件系统（已有的）

只读 · 由你维护

  • accounts/*.md — 账号定义 + 人设绑定
  • personas/*.md — 人设、风格、内容策略
  • 楼盘简介/ — 项目概况、亮点
  • 销售说辞/ — 核心卖点话术
  • 户型清单/ — 户型信息
  • 楼盘参数/ — 技术指标
  • 百问百答/ — 常见问题
  • 抗性说辞/ — 异议应对
  • 客户画像要点/ — 客群画像
  • 营销日历.md — 全年热点日历
  • media/ — 图片素材
SQLite（系统管理）

读写 · 运行时状态

  • Cookie / 登录态持久化路径
  • 发布队列 + 调度排期
  • 已发布记录 + 笔记ID
  • 发布日志 / 错误记录
  • 账号状态跟踪（active/expired/blocked）
  • 互动数据回收（点赞/收藏/评论）

内容生成流程 — 从素材到待发

单篇内容生成过程
步 阶段 输入 处理
1 读取配置 account-*.md → persona-*.md 解析 YAML，加载人设、风格、内容策略、标签池
2 选题决策 营销日历.md + 当前日期 + 项目特性 + 内容轮换策略 根据日历热点 + persona内容比例，决定今天发什么类型
3 素材加载 项目概况 + 销售说辞 + 户型清单 + 百问百答 + 图片素材 按选题类型加载对应素材文件，构建 AI Prompt 上下文
4 AI 生成 Prompt = 人设设定 + 选题 + 素材 + 风格要求 + 禁忌 Claude API 生成标题+正文+话题标签，遵循人设风格和内容禁忌
5 图片匹配 生成内容主题 + media/ 目录 + persona视觉偏好 从素材库匹配/生成配图，遵循人设的色调和排版风格
6 同项目去重 同一楼盘其他账号近期已发内容 检查相似度，确保同项目3-5个账号内容不撞车
7 排期入库 账号发布窗口 + 每日限额 + 已有排期 写入 posts 表，状态 = queued，等待发布引擎调度

关键设计决策

素材目录作为唯一真相源
系统直接从文件系统读取，不做数据导入。你更新 Markdown 后下次生成自动生效
SQLite 只存运行时状态
不复制素材内容，只存发布队列、历史、Cookie、日志等运行时数据
完整 Prompt 上下文
每篇内容生成时，AI 会收到：人设卡 + 项目素材 + 营销日历热点 + 历史去重 + 风格禁忌，确保输出质量
