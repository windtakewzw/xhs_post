账号隔离模型

Playwright BrowserContext 隔离方案
隔离层 实现 作用
BrowserContext 每个账号一个独立 Context Cookie / localStorage / 登录态完全隔离
存储目录 data/accounts/{account_id}/ 每个账号独立的 userDataDir，持久化登录态
代理IP context 级别 proxy 配置 一账号一IP（可选，按需扩展）
指纹差异 随机 viewport / locale / timezone 降低批量识别的关联风险

数据模型

SQLite Schema — accounts 表
CREATE TABLE accounts (
    id              TEXT PRIMARY KEY,          -- uuid
    name            TEXT NOT NULL,             -- 账号显示名
    project_id      TEXT NOT NULL,             -- 所属楼盘ID
    xhs_user_id     TEXT,                       -- 小红书用户ID
    login_status    TEXT DEFAULT 'pending',   -- pending/active/expired/blocked
    user_data_dir   TEXT NOT NULL,             -- Playwright userDataDir路径
    proxy_config    TEXT,                       -- JSON: {server, username, password}
    fingerprint     TEXT,                       -- JSON: {viewport, locale, timezone, ...}
    daily_limit     INTEGER DEFAULT 1,          -- 每日发布上限
    last_post_at    TEXT,                       -- 最近发布时间
    total_posts     INTEGER DEFAULT 0,
    created_at      TEXT DEFAULT (datetime('now')),
    updated_at      TEXT DEFAULT (datetime('now'))
);

登录流程

首次登录 + 登录态复用
首次登录
  1. 启动该账号的 BrowserContext
  2. 打开 xiaohongshu.com 登录页
  3. 人工扫码（等待用户操作）
  4. 检测登录成功 → 保存 Cookie
  5. 写入 login_status = active
后续复用
  1. 加载 userDataDir（含持久化Cookie）
  2. 访问首页验证登录态
  3. 有效 → 直接发布
  4. 失效 → 标记 expired，通知重新扫码
